# Financeiro Constitution

## Core Principles

### I. Agentic Framework (Model + Harness)
The "Agent = Model + Harness" formula MUST be applied to every AI interaction. The Model provides reasoning, while the Harness enforces project context, mandatory tools, and safety workflows. No agent is permitted to operate outside its defined harness.
**Rationale**: Ensures predictability and safety by bounding LLM reasoning within project-specific constraints.

### II. Domain-Driven Modular Architecture
All business logic MUST be isolated within `app/Modules/<ModuleName>` using Domain, Application, and Infrastructure layers. Controllers MUST only handle I/O and delegation; direct business rule implementation in controllers is strictly prohibited.
**Rationale**: Prevents architectural rot and promotes high cohesion and low coupling across features.

### III. Traceable Linguistic Integrity
Technical identifiers (code, file names, Git metadata) MUST be written in English (en_US). Every branch, commit, and Merge Request MUST include a numeric `TASK_ID` for full traceability. Business context and user-facing explanations MAY be in Portuguese (pt_BR).
**Rationale**: Balances global technical standards with local business needs while ensuring every change is audit-ready.

### IV. Human-in-the-Loop Financials
Automated financial operations, particularly receipt ingestion and ledger entries, MUST require explicit human confirmation. AI-driven suggestions are advisory; the final commitment to the ledger is a human responsibility.
**Rationale**: Mitigates the risk of hallucinated or incorrect financial transactions in the primary ledger.

### V. Planning-First Implementation Lifecycle
Large or high-risk tasks MUST have a technical plan generated by the Gemini Agent and approved by the user before implementation begins. This ensures architectural alignment and risk mitigation before any code is modified.
**Rationale**: Prevents wasted effort and architectural drift by aligning on the "how" before starting the "what".

## Technical Constraints
- **Primary Stack**: Laravel 11+, Vue.js 3, WAHA Gateway.
- **Environment**: Docker-standardized stack (PHP, MySQL 8, Redis).
- **Security**: HS256 JWT Authentication for all API endpoints.

## Quality & Review Workflow
- **Merge Gate**: `homolog` is the primary target for all features.
- **MR Standard**: Descriptions MUST be in Portuguese (pt_BR), include a reviewer, and be labeled `Aguardando Review`.
- **Validation**: Every task requires documented test results (manual or automated) before handover.

## Governance
The Constitution supersedes all other development practices in this repository. Amendments require documentation and a semantic version bump. Compliance review is mandatory for all Merge Requests.
